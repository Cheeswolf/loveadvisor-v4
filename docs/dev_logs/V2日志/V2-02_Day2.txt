# 📘 LoveAdvisor V2 项目开发日志 — Day 2（S2 重构与基础互动信号层重建）

## 一、今日目标

完成 LoveAdvisor V2 第二阶段重构工作，将 **S2** 从“半判断节点”重构为“基础互动信号提取器”，为后续 **S3 高级关系信号层** 与 **R1 后端规则引擎** 提供稳定输入。整体方向严格延续 V2 实施方案与 Day 1 架构冻结结果：**模型负责提信号，规则系统负责下结论，生成层负责把结论变成人话**。

本日目标包括：

* 重写 S2 Prompt
* 删除 S2 中所有高层关系判断倾向
* 增加 `interaction_reciprocity`
* 降低“全无法判断”输出概率
* 用典型样本完成第一轮稳定性回归测试

实现目标：

> 将 S2 从“容易过度保守、容易越权判断”的节点，重构为“只提低层信号、稳定可复用”的上游数据源。

---

## 二、阶段背景

在 V1 阶段，S2 虽然承担了关键信号提取功能，但同时暴露出两个核心问题：

* 上游输出过于保守，轻量模型下大量出现“无法判断”
* 节点职责过重，混入了高层关系判断倾向

这会直接导致后续模块连锁失效：
S2 一旦失真，后续判断与建议模块就整体失去可靠输入基础。相关问题已在 V1 问题清单、Day 6 日志以及 V2 方案中被明确识别，因此 V2 的首要任务之一，就是将 S2 重新定义为“纯基础互动信号层”。

---

## 三、今日核心成果

### 3.1 完成 S2 职责重定义

今日正式冻结 S2 的新职责：

* 只输出低层、可观测互动特征
* 不再判断关系阶段
* 不再输出兴趣等级
* 不再输出趋势结论
* 不再把“信息不足”直接翻译成“关系偏冷”

本次重构后，S2 的定位从：

```text
提取 + 弱判断
```

改为：

```text
基础互动信号提取
```

这一步完成后，V2 的职责分层开始真正落地。其方向与 V2 Day 1 已冻结的节点边界保持一致。

---

### 3.2 冻结新版输出协议

今日将 S2 输出协议正式确定为：

```json
{
  "initiative": "",
  "response_length": "",
  "emotional_tone": "",
  "topic_depth": "",
  "interaction_reciprocity": "",
  "key_signals": []
}
```

其中新增字段：

```text
interaction_reciprocity
```

用于描述对话承接程度，作为后续 R1 规则判断的重要输入。

这一字段的加入，使系统第一次具备了对“正向承接 / 弱承接 / 明显回避”进行显式建模的能力，严格对齐 V2 协议设计。

---

### 3.3 完成 S2 Prompt 重构

本阶段对 S2 Prompt 做了结构性升级，核心变化包括：

* 强制限定任务边界，只提基础互动信号
* 明确禁止输出“暧昧 / 拉扯 / 冷淡 / 初识 / 兴趣等级 / 升温降温”等高层结论
* 引入字段级定义与可选值范围
* 增加低信息场景特别规则
* 强制 `key_signals` 保持客观描述，不上升为关系判断

Prompt 的本质变化是：

```text
从“会做一点判断的提取节点”
→ 升级为“纯基础信号提取节点”
```

这意味着 S2 终于被压实为“数据源”而不是“解释器”。

---

### 3.4 引入“弱判断”机制

为解决 V1 中 S2 大面积“全无法判断”的问题，今日在 Prompt 中新增以下关键原则：

```text
字段独立判断
能判一点是一点
不允许整包报废
```

同时明确：

* 低信息输入不等于冷淡
* 极短对话也可输出部分弱判断
* 仅在字段确无证据时，才输出“无法判断”

这一改动的目标，不是让模型更激进，而是让上游节点从“过度保守”恢复为“可用”。这一问题在 V1 阶段已被反复验证，是本轮 S2 重构的核心修复点。

---

### 3.5 明确低信息样本处理策略

今日特别强化了对低信息场景的规则约束，包括：

* “哈哈”不应默认判冷
* “在吗 / 在”不应直接映射为关系冷淡
* 功能性短对话可部分判断，但不可上升为关系结论
* `key_signals` 必须是客观互动描述，而非关系解释

这一步是为了避免 V1 中“信息少 → 误判负面”的错误路径再次出现。

## 四、测试与验收结果

### 4.1 首轮测试样本

本轮使用以下四类样本进行首测：

| 样本 | 类型      | 测试目标         |
| -- | ------- | ------------ |
| A1 | 标准初识互动  | 验证基础互动信号是否稳定 |
| E1 | 单句低信息输入 | 验证低信息不乱判     |
| E2 | 极短功能性对话 | 验证弱判断能力      |
| D1 | 明显冷处理场景 | 验证回避信号提取能力   |

测试样本延续了既有 A / E / D 类调试资产，可直接用于 V2 上游信号层回归。

---

### 4.2 实测结果

#### 样本 A1

输出结果：

* `initiative`: A 更主动
* `response_length`: 短
* `emotional_tone`: 温
* `topic_depth`: 浅
* `interaction_reciprocity`: 正向承接

结论：

✅ 能稳定描述普通初识互动
✅ 没有误上升为高层关系判断

---

#### 样本 E1

输出结果：

* `initiative`: 无法判断
* `response_length`: 短
* `emotional_tone`: 温
* `topic_depth`: 无法判断
* `interaction_reciprocity`: 无法判断

结论：

✅ 已明显摆脱“整包全无法判断”
✅ 能对单句低信息内容做最小有效识别
⚠️ `topic_depth` 后续可进一步收敛为“浅”

---

#### 样本 E2

输出结果：

* `initiative`: A 更主动
* `response_length`: 短
* `emotional_tone`: 温
* `topic_depth`: 浅
* `interaction_reciprocity`: 弱承接

结论：

✅ 已具备低信息功能性对话下的弱判断能力
✅ 不再把极短对话直接误翻成“冷”

---

#### 样本 D1

输出结果：

* `initiative`: A 更主动
* `response_length`: 短
* `emotional_tone`: 冷
* `topic_depth`: 浅
* `interaction_reciprocity`: 明显回避

结论：

✅ 能稳定抓住“拒绝继续聊天”这类表层回避信号
✅ 没有越权直接输出“冷淡期”

---

### 4.3 验收标准与结果

| 验收项                            | 结果 | 评价                    |
| ------------------------------ | -- | --------------------- |
| 输出是否稳定                         | 通过 | 四条样本均为稳定 JSON         |
| 是否去掉高层判断                       | 通过 | 未出现暧昧/拉扯/冷淡等结论        |
| 是否减少整包“无法判断”                   | 通过 | E1、E2 表现明显改善          |

## 五、遗留优化点

本轮仅发现一处轻微可优化问题：

| 问题                                | 说明          | 优化方向                            |
| --------------------------------- | ----------- | ------------------------------- |
| 单句低信息样本中 `topic_depth` 偶尔输出“无法判断” | 表现可接受，但不够收敛 | 补充规则：`哈哈 / 嗯 / 哦` 等极短语气词优先归入“浅” |

该问题不影响 Day 2 整体验收，仅属于字段收敛性优化。
