# 📘 LoveAdvisor 项目背景文档（V1 → V2.5 全阶段整合版）

---

# 一、项目概述

## 1.1 项目目标

LoveAdvisor 是一个基于对话内容的情感分析与决策辅助系统，核心目标是：

> **从聊天文本中识别关系阶段 + 兴趣程度，并生成克制、合理的行动建议**

系统核心输出包括：

* relationship_stage（关系阶段）
* interest_level（兴趣等级）
* key_signals（关键信号）
* psychological_analysis（心理分析）
* risk_points（风险点）
* suggestions（建议）
* next_step（下一步动作）

---

## 1.2 核心问题

项目要解决的本质问题是：

| 层级  | 问题                |
| --- | ----------------- |
| 识别层 | 如何从非结构化聊天中提取关系信号  |
| 判断层 | 如何稳定判断关系阶段与兴趣强度   |
| 决策层 | 如何避免“过度推进”，输出克制建议 |
| 工程层 | 如何让系统稳定、可测、可优化    |

---

## 1.3 方法论核心

项目最终形成的核心方法论：

> **规则主导决策（R1） + 模型负责表达（S5）**

即：

* 关键判断由规则系统完成（可控、稳定）
* 模型只负责生成自然语言表达（不参与决策）

---

# 二、V1 阶段（原型探索期）

## 2.1 阶段目标

* 验证“聊天分析 + 恋爱建议”是否可行
* 初步构建分析链路

## 2.2 系统特征

* 单链路 LLM 输出（强依赖模型）
* 无明确结构拆分
* 无规则系统

## 2.3 主要问题

* 输出高度不稳定
* 无法复现结果
* 建议过度主观、容易“劝进”

## 2.4 阶段结论

> ❗ 纯模型方案不可控，必须引入结构化与规则系统

---

# 三、V2 阶段（结构化与规则化）

## 3.1 阶段目标

* 拆解分析链路
* 引入信号层 + 规则层

---

## 3.2 核心架构（V2）

```text
输入文本
→ S2（基础信号提取）
→ S3（高级信号识别）
→ R1（规则判断）
→ S5（建议生成）
→ 输出
```

---

## 3.3 核心创新

### ① 信号分层

* S2：互动行为（主动性、长度、情绪等）
* S3：关系信号（暧昧、拒绝、拉扯等）

---

### ② R1规则系统（核心）

首次引入：

```text
relationship_stage
interest_level
```

由规则推导，而非模型生成。

---

### ③ 建议层与判断层解耦

* R1：决定“是什么状态”
* S5：决定“怎么说”

---

## 3.4 阶段成果

* 初步实现结构化分析
* 建立关系阶段体系：

| 阶段   |
| ---- |
| 初识期  |
| 暧昧期  |
| 拉扯期  |
| 冷淡期  |
| 无法判断 |

---

## 3.5 核心问题

* 测试依赖人工
* 规则调整无量化标准
* 边界（尤其E类）失控
* interest_level不稳定

---

## 🟢 Day 4：E类边界修复（关键突破）

### 核心改动：

引入：

```text

```

---

### 实现：

* 极弱互动识别
* 功能性对话识别
* 拒判机制

---

### 成果：

* E类准确率：0% → 100%
* stage_accuracy → 1.0
* r1_strict_accuracy → 0.8

---

### 意义：

> 系统具备“拒判能力”，进入成熟规则系统阶段

---

## 🟢 Day 5：interest_level 精修

### 核心优化：

* 初识期：默认低，限制中档触发
* 暧昧期：收紧高档条件
* 拉扯期：建立低/中分流

---

### 当前结果：

* r1_strict_accuracy ≈ 0.67
* 问题集中在 A/B/C 边界

---

### 阶段结论：

> 系统进入“精细规则调参阶段”
