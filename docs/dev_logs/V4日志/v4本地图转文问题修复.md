# 一、MTU目标回顾

本任务核心目标：

```text
将 I1 从“图转文”升级为“图转文 + 轻结构化聊天草稿生成”
````

具体要求：

* 自动区分说话人（A/B 或 我/对方）
* 恢复对话轮次
* 提供可直接用于分析的草稿
* 保持 I1/I2/analyze 架构边界不变

# 二、修改内容总览

## 修改文件列表

* app/schemas/image_to_text_models.py
* app/services/image_to_text_service.py
* web/lib/api.ts
* web/app/page.tsx

---

## 核心改动

### 1️⃣ 输出协议升级

新增字段：

```json
structured_chat_draft: string
```

字段作用：

```text
带说话人区分与轮次结构的聊天草稿（默认输入）
```

同时保留：

* raw_text（原始识别）
* merged_text（普通整理）

---

### 2️⃣ I1 Prompt 升级

从：

```text
提取图片中的文本
```

升级为：

```text
提取聊天内容，并恢复说话人结构与对话轮次
```

模型职责变化：

```text
纯OCR → 轻结构化生成
```

---

### 3️⃣ Service 层逻辑

新增：

* structured_parts 收集结构化文本
* structured_chat_draft 聚合输出

特点：

```text
多图 → 合并结构化草稿
去除图片前缀
保持顺序一致
```

---

### 4️⃣ 前端回填策略优化

```ts
const draftText = response.structured_chat_draft || response.merged_text;
```

逻辑：

1. 优先使用结构化草稿
2. 兜底普通草稿
3. 用户仍可编辑确认
