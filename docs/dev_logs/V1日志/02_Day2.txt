## 二、核心改动内容

### 2.1 引入「可判性判断层（Judgable Layer）」

#### 问题背景

早期版本存在严重问题：

* 单句也强行判断
* 功能性对话误判为初识期
* 浅层互动误判为冷淡期

#### 解决方案

新增最高优先级规则：

```text
先判断：是否可判断
再判断：关系阶段
```

#### 判定标准（核心）

* 单句 / 极少信息 → 不可判
* 功能性对话（在吗→在） → 不可判
* 单轮浅层问答 → 不可判
* 无情绪 / 无关系信号 → 不可判

#### 结果

✅ 完全解决：

* E1 / E2 / E3 全部误判问题
* 模型“过度推理”问题

---

### 2.2 重构 relationship_stage 判定逻辑（优先级化）

#### 原问题

* 规则冲突（暧昧 vs 拉扯 vs 冷淡）
* 模型判断顺序混乱

#### 优化结构（强制顺序）

```text
可判性 → 暧昧 → 拉扯 → 冷淡 → 初识
```

#### 关键改动

| 模块  | 优化点           |
| --- | ------------- |
| 暧昧期 | 必须有明确情绪拉近     |
| 拉扯期 | 必须“推进 + 回避”共存 |
| 冷淡期 | 必须“持续性”信号     |
| 初识期 | 作为兜底，而非默认     |

#### 结果

✅ 阶段判断稳定性显著提升
✅ 不再出现“随便归类初识期”的问题

---

### 2.3 明确「冷淡期 vs 初识期」边界

#### 核心问题

模型曾频繁混淆：

* 冷 → 冷淡期 ❌
* 浅 → 初识期 ❌

#### 关键规则补丁

```text
冷淡期必须满足：
- 持续低投入
- 明显降温
- 非单轮
```

#### 核心认知

```text
单次冷 ≠ 冷淡期
持续冷 → 冷淡期
```

#### 结果

✅ D类样本判断稳定
✅ 不再误判“短对话为冷淡”

---

### 2.4 重构 interest_level 判定体系

#### 原问题

* 把“单方热情”当成高
* 无法区分：

  * 中性回避
  * 负面回避

#### 优化核心

### ⭐ interest_level = 关系推进能力（非单方意愿）

---

#### 判定逻辑

| 情况           | 结果 |
| ------------ | -- |
| 礼貌互动 / 信息交换  | 低  |
| 一方推进，对方未拒绝   | 中  |
| 一方推进，对方冷淡/否定 | 低  |
| 双方推进         | 高  |

---

#### 关键补丁

```text
简短回应 = 中性
冷淡表达 = 负面（优先级更高）
```

#### 结果

✅ 修复 C2 / C3 样本误判
✅ interest_level 精度显著提升

---

### 2.5 引入「联动规则」

```text
若 relationship_stage = 无法判断
→ interest_level 必须 = 无法判断
```

#### 解决问题

* E类样本中“强行给低”的错误

#### 结果

✅ 完全消除“半判断状态”
