## 二、完成模块

### 2.1 S1：文本预处理（稳定版）

#### 功能

将原始聊天记录转换为结构化对话：

```
A: ...
B: ...
A: ...
```

#### 关键优化

| 问题                   | 解决方案               |
| -------------------- | ------------------ |
| 误删“哈哈”等情绪内容          | 定义“低信息 ≠ 无信息”      |
| 输出 reasoning_content | 强制只输出 cleaned_chat |
| 空文本处理混乱              | 增加输入有效性判断          |

#### 当前能力

* 支持单句输入
* 支持低信息文本（哈哈 / 嗯）
* 支持去噪（链接/系统信息）

---

### 2.2 S2：行为信号提取（核心模块）

#### 输出结构

```json
{
  "initiative": "",
  "response_speed": "",
  "response_length": "",
  "emotional_tone": "",
  "topic_depth": "",
  "key_signals": [],
  "emotional_stability": ""
}
```

---

#### 核心能力拆解

| 能力            | 当前状态         |
| ------------- | ------------ |
| 主动性识别         | ✅ 稳定         |
| 情绪判断（冷/温/热）   | ✅ 稳定         |
| 暧昧识别          | ✅ 已支持        |
| 情绪压力识别        | ✅ 已支持        |
| 互动结构理解        | ✅ 已支持        |
| topic_depth判断 | ⚠️ 已修正（避免低估） |

---

#### 关键规则沉淀

**1. 情绪优先级规则**

```
情绪表达 > 互动结构
```

避免：

* “哈哈”被判为冷
* “暧昧”被判为温

---

**2. 情绪分类逻辑**

| 类型             | 判定       |
| -------------- | -------- |
| 暧昧 / 撩 / 梦到你   | 热        |
| 正常互动 + 正向反馈    | 温        |
| 冷处理 / 回避 / 无延展 | 冷        |
| 情绪压力 / 质问      | 冷（优先级最高） |

---

**3. 深度判断逻辑**

| 层级 | 内容       |
| -- | -------- |
| 浅  | 日常信息     |
| 中  | 情绪互动     |
| 深  | 认知/价值/未来 |

---

**4. 低信息处理**

| 输入  | 处理          |
| --- | ----------- |
| 哈哈哈 | 温 + 浅       |
| 嗯   | 冷 或 温（视上下文） |
| 空   | 全部“无法判断”    |

---

---

### 2.3 S3：情绪趋势判断（初版）

#### 输出结构

```json
{
  "emotional_trend": "",
  "confidence": ""
}
```

---

#### 核心认知

```
S2 = 状态识别
S3 = 变化判断
```

---

#### 判断规则

| 情况    | 输出   |
| ----- | ---- |
| 无变化证据 | 平稳   |
| 明显变好  | 升温   |
| 明显变差  | 降温   |
| 无数据   | 无法判断 |

---

#### 关键修复

| 问题          | 修复       |
| ----------- | -------- |
| 默认输出“平稳”    | 改为“无法判断” |
| 置信度滥用       | 无依据时必须低  |
| 把“普通聊天”误判降温 | 增加对比逻辑   |
