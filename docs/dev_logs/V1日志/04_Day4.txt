## 四、关键工程设计（本阶段核心）

---

### 4.1 JSON 强结构约束

* 所有字段必须存在
* 禁止新增字段
* 禁止修改字段名
* 禁止输出解释性内容

---

### 4.2 缺失数据处理机制（防崩核心）

| 类型    | 处理方式     |
| ----- | -------- |
| 字符串字段 | `"无法判断"` |
| 数组字段  | `[]`     |

---

### 4.3 输入异常处理（字符串注入适配）

由于 Coze 输入为字符串：

```text
S2_output / S3_output / S4_output / S5_output
均为 JSON 字符串
```

处理规则：

```text
先解析 → 再映射
解析失败 → 按缺失处理
```

---

### 4.4 联动规则（继承 S4）

```text
若 relationship_stage = 无法判断
→ interest_level 必须 = 无法判断
```

---

### 4.5 禁止行为（重要）

```text
❌ 不允许重新推理
❌ 不允许修改 S5 文本
❌ 不允许补充主观判断
```
