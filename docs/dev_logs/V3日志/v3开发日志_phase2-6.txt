# 三、关键工程问题记录（重点）

---

## ❗问题1：Token 上下文溢出（核心问题）

### 表现

```
maximum context length exceeded
```

### 原因

* 长对话历史
* 大量项目背景注入
* 高 completion 设置

### ✅ 解决方案

#### 开发模式转变：

```text
聊天式开发 → 任务分片式开发
```

#### 强制规范：

* 每阶段新开聊天
* 仅传最小任务上下文
* 单任务 token ≤ 2k

---

## ❗问题2：枚举值不统一

### 表现

* "A 更主动" vs "A更主动"

### 风险

* parser 误判
* 规则降级为“无法判断”

### ✅ 解决方案

* 统一标准值
* parser 兼容映射
* schema 强约束

---

## ❗问题3：职责边界漂移风险

### 表现

* S2/S3 想做判断
* S5 想做阶段推理

### ✅ 解决方案

明确架构原则：

```text
S2/S3：信号层
R1：决策层
S5：表达层
Guardrail：约束层
```
