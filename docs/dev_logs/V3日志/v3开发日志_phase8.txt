## 三、关键开发内容

---

### 1️⃣ 编码与请求链路修复（Phase 8.1）

**问题：**

* 中文乱码（GBK / UTF-8 混用）
* data发送JSON导致解析异常

**修复：**

```python
requests.post(API_URL, json=payload)
```

**结果：**

* 中文链路完全稳定
* payload → API → S2 无损传递

---

### 2️⃣ Pipeline异常可观测性（Phase 8.1+）

新增：

* `pipeline_error_stage`
* `pipeline_error_message`
* `traceback_summary`
* 保留 S2 / S3 / R1 中间结果

👉 解决问题：

> ❌ 出错只看到“系统异常”
> ✅ 能定位具体挂在哪一层

---

### 3️⃣ S5 user_question 依赖解除

**问题：**

* 空 user_question 直接抛异常 → 全链路兜底

**修复：**

```python
user_question = "基于当前聊天情况，我应该如何推进关系？"
```

**结果：**

* S5稳定运行
* 不再触发系统兜底

---

### 4️⃣ R1 可判性体系重构（核心）

重构函数：

* `is_very_weak_s2`
* `is_functional_minimal_interaction`
* `is_judgable`

核心思想：

> ❗从“结构判断” → “信息密度判断”

---

### 5️⃣ E组边界修复（多轮迭代）

问题：

| 样本 | 错误    |
| -- | ----- |
| E2 | 初识期误判 |
| E3 | 初识期误判 |

最终解决方案：

```python
if 极弱特征组合 + 全部低信息信号:
    return "无法判断"
```

**结果（最终）：**

| 样本 | 状态           |
| -- | ------------ |
| E1 | ✅ 正确         |
| E2 | ✅ 修复         |
| E3 | ⚠️ 仍存在1例边界失守 |

---

### 6️⃣ C / D 组结构修复

#### C1（拉扯 → 冷淡误判）

修复：

* 冷淡期判定收紧
* 必须“持续冷淡 + 多信号确认”

#### D1（冷淡 → 拉扯误伤）

修复：

* 拉扯期需 ≥2关系信号

### 7️⃣ R1 Debug体系建设（关键突破）
