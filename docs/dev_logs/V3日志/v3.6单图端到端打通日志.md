## 五、阶段问题与修复路径

### 1️⃣ OCR 技术路线错误（关键转折点）

**问题：**

```text
传统 OCR 无法生成可用聊天草稿
```

**修复：**

```text
切换 Provider → Qwen-OCR（视觉大模型）
```

---

### 2️⃣ I1 返回结构问题

**问题：**

```text
content 类型不稳定（list / str）
```

**修复：**

```python
if isinstance(content, list):
    content = "".join([item.get("text", "") for item in content])
```

---

### 3️⃣ I1 → I2 回填失败

**问题：**

```text
merged_text 未显示在输入框
```

**根因：**

```text
Streamlit text_area 使用 value 而非 key 绑定
```

**修复：**

```python
key="transcription_text"
```

---

### 4️⃣ 前端 provider 错误

**问题：**

```text
前端默认 provider = vision_model_v1
```

**修复：**

```text
统一切换为 qwen_ocr
```

---

### 5️⃣ Analyze provider 未对齐（关键问题）

**问题：**

```text
前端显示 deepseek
后端实际使用 mock
```

**根因：**

```text
st.session_state.get('provider', 'mock') 默认回退
+ OCR provider 变量污染
```

**修复：**

```python
provider = st.session_state.provider
```

并进行变量隔离：

```text
provider（LLM）
ocr_provider（OCR）
```

---

### 6️⃣ “网络错误”误判问题

**问题：**

```text
前端提示网络错误，但后端返回 200
```

**结论：**

```text
非真实网络错误
而是 provider 未对齐 + 状态混乱导致的假异常
```
