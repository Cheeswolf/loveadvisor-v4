# 📘 LoveAdvisor V3 稳定版开发日志（正式归档版）

---

# 一、项目概述

**项目名称**：LoveAdvisor
**版本阶段**：V3 稳定版（Stabilized Release）
**开发周期**：V1 → V2 → V2.5 → V3（稳定化阶段）
**核心目标**：

> 构建一个基于对话文本的「关系阶段 + 兴趣等级 + 策略建议」分析系统，实现稳定、可解释、可落地的情感决策支持能力。

---

# 二、V3阶段核心升级

相较于 V2 / V2.5，V3 的本质升级在于：

## 1️⃣ 架构层升级

构建完整分析链路（Pipeline）：

```text
PREPROCESS
→ S2 基础信号提取
→ S3 高级信号提取
→ R1 规则推断（阶段 + 兴趣）
→ S5 策略生成
→ GUARDRAIL
→ OUTPUT_BUILDING
```

---

## 2️⃣ 规则系统升级（R1）

建立结构化规则体系：

* relationship_stage（关系阶段）
* interest_level（兴趣等级）
* 多维信号融合（S2 + S3）
* 可解释推断路径（debug可追踪）

---

## 3️⃣ 测试体系升级

引入完整评测系统：

* 批量测试（15标准样本）
* 指标体系：

  * stage_accuracy
  * r1_strict_accuracy
  * suggestion_alignment_rate
  * next_step_conservative_rate
* 问题分类机制（低估 / 高估 / 边界失守等）

---

# 三、稳定化阶段（核心阶段）

V3 的关键不在“开发”，而在：

> ❗**稳定化（Stabilization）**

---

## 1️⃣ 稳定化目标

解决三类核心问题：

| 问题类型             | 表现               |
| ---------------- | ---------------- |
| interest_level漂移 | 中/高误判            |
| 边界不稳             | 无法判断 vs 初识期混淆    |
| 系统一致性问题          | debug vs 实际结果不一致 |

---

## 2️⃣ 方法论（关键）

V3 稳定化采用：

```text
误差簇驱动 + 单点修复 + 回归验证
```

核心原则：

* 一次只改一个模块（MTU）
* 不跨层修改
* 必须回归测试
* 用数据驱动，而不是主观判断

---

# 四、关键问题与解决过程

---

## 🟢 问题1：初识期中兴趣低估（A2/A3）

### 现象

```text
A2 / A3：期望 中 → 实际 低
```

---

### 根因分析

* S2/S3 信号正常
* `_initial_medium_signal` 计分不足
* weak_interaction（短+弱承接）强拦截

---

### 解决路径

#### Step 1：错误尝试（无效）

* 提升分数权重 → ❌ 无效
* 条件补丁 → ❌ 逻辑不成立

#### Step 2：根因定位

* 确认拦截点：`weak_interaction`

#### Step 3：最终修复（关键）

```text
新增补偿分支：
weak_interaction + has_positive_reciprocity → 放行
```

---

### 中间问题（关键突破）

发现：

```text
规则返回 = 中
API输出 = 低
```

👉 最终定位：

> ❗API未加载最新代码（运行环境问题）

---

### 最终结果

```text
A1 → 低 ✅
A2 → 中 ✅
A3 → 中 ✅
```

👉 **初识期完全收敛**

---

# 九、关键经验总结（非常重要）

---

## 1️⃣ 不要把“规则问题”和“运行问题”混淆

A2/A3 最大突破：

```text
不是规则错
而是服务没加载新代码
```

---

## 2️⃣ 稳定化 ≠ 写提示词

本阶段本质是：

```text
调系统行为函数
```

---

## 3️⃣ 必须用“误差簇驱动”

不是改功能，而是：

```text
针对具体case → 定点修复 → 回归验证
```

---

## 4️⃣ 不追求100%正确

正确目标是：

```text
整体收敛 + 可控误差
```
