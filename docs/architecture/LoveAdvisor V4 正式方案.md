# 二、系统总体架构（5 层）

---

## 2.1 架构总览

```text
1. 用户入口层
2. 输入处理层
3. 核心分析引擎层
4. 产品数据层

```


```text
PREPROCESS → S2 → S3 → R1 → S5 → GUARDRAIL → OUTPUT
```

### 产品数据层（新增）

* users
* analysis_records
* usage_quotas
* feedback_records

### 商业化预留层（不实现）

* 套餐
* 支付

# 三、冻结边界（强制执行）

## 3.3 开放区（允许新增）

```text

```

---

# 四、执行体系（核心新增）

---

## 4.1 执行模型

```text
V4 = Phase（阶段） → MTU（最小任务单元） → Claude Code执行
```

---

## 4.2 Phase 划分

```text
Phase 1：输入闭环
Phase 2：展示闭环
Phase 3：数据闭环
Phase 4：产品增强
Phase 5：商业化预留
```

---

## 4.3 MTU 规则（强制）

```text
1. 一次只做一个任务
2. 只修改一个模块
3. 必须可运行
4. 必须可验收
5. 未验收禁止进入下一任务
```

---

## 4.4 Claude Code 使用纪律

```text
- 禁止跨模块修改
- 禁止顺手优化
- 禁止修改 analyze / rules
- 只允许改指定文件
- 每次必须输出修改清单
```

---

## 4.5 标准执行流程

```text
任务定义 → MTU任务单 → Claude Code执行 → 验收 → 下一MTU
```

---

## 4.6 错误处理机制

```text
新问题 = 新MTU
禁止在当前任务顺手修复
```

---

# 🧩 Phase 1：输入闭环（最高优先级）

## 目标

```text
让用户能从截图/文本顺利进入 analyze
```

---

## MTU任务树

### MTU-1：前端截图上传组件

* 文件：frontend
* 功能：上传图片 + 预览

---

### MTU-2：image-to-text 空接口

* 文件：app/api/image_to_text.py
* 返回 mock 数据

---

### MTU-3：I1 Service 接入 Qwen-OCR

* 文件：image_to_text_service.py
* 输出 raw_text + merged_text
* 使用 qwen_ocr（见补丁方案） 

---

### MTU-4：前端接入 I1

* 上传 → 调接口 → 填入 I2 编辑区

---

### MTU-5：I2确认联动

* 编辑文本 → 点击分析 → 调 /analyze

---

## Phase 验收

