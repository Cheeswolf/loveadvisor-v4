# 📘 LoveAdvisor 网页版发布方案（试点版 · 可执行）

---

# 一、发布目标（必须统一认知）

## 🎯 本次发布定位

```text
不是正式上线，而是试点版发布（MVP验证）
````

---

## ✅ 成功标准

```text
1. 用户能打开链接
2. 能输入（文本 / 截图）
3. 能完成分析
4. 能看懂结果
5. 愿意再用一次（核心指标）
```

---

# 二、发布架构方案（推荐）

## 🧩 前后端分离部署（标准方案）

```text
用户浏览器
↓
Vercel（前端 · Next.js）
↓
Render / Railway（后端 · FastAPI）
↓
分析引擎（pipeline）
```

---

## ✅ 优点

* 部署快（1–2小时内上线）
* 成本低（免费层可用）
* 支持快速迭代
* 符合现有 V4 架构

---

## ❌ 不推荐方案

```text
前后端一体部署（FastAPI托管前端）
```

原因：

* 不利于前端演进
* 后续必拆
* 体验差

---

# 四、部署流程（逐步执行）

---

# 🧩 Step 1：后端整理

## 🎯 目标

```text
后端可在服务器独立运行
```

---

## 必须满足

### 1️⃣ 启动命令

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

---

### 2️⃣ requirements.txt 完整

必须包含：

* fastapi
* uvicorn
* requests / httpx
* pydantic
* 其他依赖

---

### 3️⃣ 环境变量

```text
DEEPSEEK_API_KEY=xxx
DASHSCOPE_API_KEY=xxx
```

---

---

# 🧩 Step 2：部署后端（Render）

---

## 操作流程

### 1️⃣ 推送 GitHub

```bash
git init
git add .
git commit -m "deploy"
git remote add origin xxx
git push -u origin main
```

---

### 2️⃣ 创建服务

```text
Render → New Web Service → 选择仓库
```

---

### 3️⃣ 配置

```text
Build Command:
pip install -r requirements.txt

Start Command:
uvicorn app.main:app --host 0.0.0.0 --port $PORT
```

---

### 4️⃣ 设置环境变量

```text
DEEPSEEK_API_KEY
DASHSCOPE_API_KEY
```

---

### 5️⃣ 获取API地址

```text
https://xxx.onrender.com
```

# 🧩 Step 5：配置 CORS（必须）

---

## FastAPI 增加

```python
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # 试点阶段全开
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```
