# 📘 LoveAdvisor V4 本地优先联调与发布前收口方案（重写版）

---

## 一、方案定位

### 1.1 本次重写的原因

当前推进过程中，已经暴露出一个关键问题：

```text
云端部署问题、前后端联调问题、本地代码问题，被混在了一起。
```

这会导致：

* 排查对象混乱
* 修改成本上升
* 每次修复都要重复上传云端
* 很难判断问题到底属于代码层还是部署层

因此，原来的“边上云边排查”路径需要调整。

---

### 1.2 新的核心决策

```text
先把本地版本完整跑通，再上传云端。
```

也就是说：

* **本地环境** = 唯一开发与联调基线
* **云端环境** = 本地验证完成后的发布环境

---

### 1.3 本阶段目标

本阶段只做两件事：

1. 建立一个**本地稳定可复现的产品闭环版本**
2. 在本地闭环稳定后，再进行一次性云端发布

---

## 二、阶段定义

建议将当前阶段正式命名为：

```text
LoveAdvisor V4.1：本地优先联调与发布前收口阶段
```

这个命名强调的是：

* 不再以云端排查为主
* 不再继续扩功能
* 不再把部署问题和产品问题混在一起
* 先完成本地“可用产品版”的验证

---

## 三、核心原则

---

### 3.1 原则一：本地是唯一真基线

从现在开始，所有功能排查、联调、体验验证，都以本地环境为准。

正式定义：

```text
本地前端 → 本地后端
```

禁止默认采用：

```text
本地前端 → 云端后端
```

作为主调试模式。

---

### 3.2 原则二：先验证产品闭环，再验证部署闭环

本阶段首先要证明的是：

```text
产品逻辑成立
```

而不是：

```text
云端平台配置是否成立
```

所以优先级必须是：

1. 本地功能闭环
2. 本地用户路径闭环
3. 本地体验验证
4. 云端部署验证

---

### 3.3 原则三：问题必须分层处理

所有问题必须明确归属到以下四层之一：

#### A. 前端交互层

例如：

* 上传不触发
* 按钮不可用
* 状态显示错误
* 结果页不渲染

#### B. 接口联调层

例如：

* 前端未发请求
* 请求路径错误
* 协议字段不一致
* 返回结构异常

#### C. 后端能力层

例如：

* analyze 报错
* image-to-text 报错
* provider 调用失败
* 环境变量缺失

#### D. 部署层

例如：

* Render 服务未更新
* Vercel 环境变量错误
* CORS 问题
* 公网链路异常

本阶段只处理 A / B / C，暂不把 D 作为主战场。

---

### 3.4 原则四：本阶段禁止继续扩功能

本阶段不是继续做“新能力”，而是把已有能力彻底跑通。

因此明确禁止：

* 重构 analyze
* 重构 rules
* 重构 pipeline
* 新增商业化模块
* 新增登录/支付
* 再开新前端形态
* 继续做复杂 UI 扩展

---

### 3.5 原则五：必须采用 MTU 纪律

所有施工继续采用：

```text
MTU = 最小任务单元
```

每次只允许：

* 解决一个问题
* 修改一个模块
* 形成一次可验证结果

禁止：

* 多问题混改
* 顺手优化
* 连续不验收推进

---

## 四、本阶段成功标准

只有满足以下条件，才允许进入云端发布阶段：

### 4.1 文本输入链路通过

```text
文本输入 → analyze → 结果展示
```

必须本地稳定可用。

---

### 4.2 截图输入链路通过

```text
截图上传 → image-to-text → I2确认 → analyze → 结果展示
```

必须本地稳定可用。

---

### 4.3 完整用户路径通过

```text
输入 → 分析 → 理解结果 → 查看下一步 → 历史 → 再次分析
```

必须本地可完整走通。

---

### 4.4 联调结果可复现

同样的本地启动方式、同样的环境变量、同样的测试路径，必须可重复成功。

---

### 4.5 本地问题与部署问题彻底分离

必须形成明确状态：

```text
本地版本已通过
```

之后如果云端出问题，就只归类为部署问题，不再怀疑产品主链路。

## 十、最终结论

本次方案重写后的核心路线只有一句话：

```text
先本地闭环，再云端发布。
```

具体执行顺序固定为：

```text
本地运行基线
→ 文本链路打通
→ 截图链路打通
→ 完整用户路径验证
→ 本地候选版冻结
→ Render + Vercel 云端部署
→ 小范围试用
```

如果本地没有形成稳定基线，就不再继续以云端作为主排查场。
